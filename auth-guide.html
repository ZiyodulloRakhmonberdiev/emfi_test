Подробная инструкция по авторизации в amoCRM

Важно: Если вы получаете ошибки 400 или 401, скорее всего проблема в авторизации. Следуйте этой инструкции шаг за шагом.

Получение Authorization Code

1 Войдите в свой аккаунт amoCRM

Перейдите на сайт amocrm.ru и войдите в свой аккаунт.

2 Перейдите в настройки

Нажмите на свой профиль в правом верхнем углу и выберите "Настройки".

3 Перейдите в раздел интеграций

В левом меню выберите "API" и затем "Интеграции".

4 Создайте новую интеграцию

Нажмите кнопку "Добавить интеграцию" и заполните следующие поля:

  • Название: Любое название, например "Тестовое приложение"
  • Redirect URI: https://example.com (важно указать именно этот URL)
  • Права доступа: Выберите все необходимые права (минимум: "Просмотр сделок", "Просмотр контактов", "Просмотр задач")

Нажмите "Сохранить".

5 Получите данные интеграции

После создания интеграции вы увидите следующие данные:

  • Integration ID - это ваш Client ID
  • Secret Key - это ваш Client Secret

Сохраните эти данные, они понадобятся для авторизации.

6 Получите Authorization Code

Для получения Authorization Code необходимо выполнить запрос авторизации. Самый простой способ:

  1. Перейдите во вкладку "Ключи и доступы" вашей интеграции
  2. Найдите раздел "Authorization code" и нажмите кнопку "Получить код"
  3. Система выдаст вам код авторизации, который действует 20 минут
Внимание: Authorization Code действителен только 20 минут! Если вы не успеете использовать его за это время, придется получить новый код.

Получение Access Token

1 Запустите прокси-сервер

Перед началом работы с приложением необходимо запустить прокси-сервер:

node proxy.js

Вы должны увидеть сообщение:

CORS Proxy server running at http://localhost:8080

2 Введите данные в приложение

В приложении введите следующие данные:

  • Client ID: Integration ID из вашей интеграции
  • Client Secret: Secret Key из вашей интеграции
  • Authorization Code: Код, полученный на предыдущем шаге
  • Redirect URI: https://example.com (должен совпадать с URI, указанным при создании интеграции)
  • Поддомен: ziyodullorakhmonberdiev (или ваш поддомен amoCRM)

3 Получите Access Token

Нажмите кнопку "Получить Access Token". Если все данные введены правильно, вы получите токен доступа, который будет автоматически сохранен и вставлен в поле "Access Token".

Совет: Если вы видите ошибки в консоли браузера (F12), они могут содержать дополнительную информацию о проблеме.

4 Загрузите данные

После получения токена нажмите кнопку "Загрузить сделки" для получения данных из amoCRM.

Решение проблем

Ошибка 400 (Bad Request)

Возможные причины:

  • Неверный формат данных в запросе
  • Неверный Redirect URI (должен точно совпадать с URI, указанным при создании интеграции)
  • Истекший Authorization Code (действует только 20 минут)

Решение: Получите новый Authorization Code и убедитесь, что все данные введены правильно.

Ошибка 401 (Unauthorized)

Возможные причины:

  • Неверный Client ID или Client Secret
  • Истекший Access Token
  • Неверный формат токена в заголовке Authorization

Решение: Проверьте правильность Client ID и Client Secret, получите новый токен доступа.

Совет: Обновленный прокси-сервер теперь выводит подробную информацию о запросах и ответах в консоль. Это поможет вам диагностировать проблемы.
